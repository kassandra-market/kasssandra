## Tasks

### RF1 — Finalize / crank flows (action + gated E2E + crank UI)
- **Actions** (`app/src/data/actions/finalize.ts`): `buildFinalizeProposalsIxs`, `buildAdvancePhaseIxs`, `buildFinalizeFactsIxs`, `buildFinalizeAiClaimsIxs`, `buildFinalizeOracleIxs` — each assembles the SDK builder with the oracle + the child tail (proposers/facts pubkeys from `fetchOracleDetail`) + any mint/vault accounts. For the FULL-set finalizes (finalizeProposals/finalizeOracle) that overflow a legacy tx near MAX_PROPOSERS=60, use `sendFinalizeViaAlt` (v0/ALT) when the tail is large (the send seam must accept a v0-capable sender OR the action returns a marker that the UI sends via ALT — design a clean way; the E2E can drive the ALT path for a large set). These are crank-like (may be permissionless — the signer is just the fee-payer; confirm which need a specific authority).
- **Offline unit tests** + **gated surfpool E2E** (`app/test/finalize.e2e.test.ts`): seed an oracle with proposers/facts, drive finalizeProposals → assert phase advanced (FactProposal); submitFact/advance → finalizeFacts → assert; finalizeAiClaims + finalizeOracle → assert **Resolved** with the resolved option. (Reuse the harness; mirror the lifecycle/settlement E2E progression.) Prove the near-cap v0/ALT finalize path if tractable (or document it's exercised by the SDK's own v0-alt E2E + unit-covered here).
- **Crank UI:** extend `OracleActions` — in each pre-Resolved phase, a "crank/finalize" control (a PrimaryChestnut button, e.g. "Finalize proposals", "Advance phase") that runs the finalize action via the wallet sender + the status UX + refetch. Permissionless-friendly (anyone connected can crank). Delphi + a11y + phase-gated.
- Headless-render the crank states. `pnpm --filter app typecheck`+`test`+`build`(verify-css)+`lint` green; RUN the gated finalize E2E. Commit `feat(app): finalize/crank write flows + gated E2E`.

### RF2 — Claim / close / sweep flows (action + gated E2E + payout UI)  [after RF1]
- **Actions** (`app/src/data/actions/claims.ts`): `buildClaimProposerIxs`, `buildClaimFactIxs`, `buildClaimFactVoteIxs` (per-participant — the authority claims ITS proposer/fact/vote; derive the authority's KASS ATA + idempotent-create), `buildCloseAiClaimIxs`, `buildCloseMarketIxs`, `buildSweepOracleIxs` (sweep → dao_treasury = ATA(dao_authority, kass_mint); needs governance set). Validation + the right rent/recipient accounts.
- **Offline unit tests** + **gated surfpool E2E** (`app/test/claims.e2e.test.ts`): drive an oracle to Resolved (reuse RF1's progression), then claimProposer/claimFact/claimFactVote via the participant keypair → assert the KASS payout landed + the accounts closed; closeAiClaim/closeMarket + sweepOracle (after grace + governance) → assert. (Mirror `settlement-e2e.test.ts`.)
- **Payout UI:** in the Resolved/terminal phase, on each Proposer/Fact/AI-claim/Market card, a "Claim"/"Close" control gated on the connected wallet being that participant (authority == publicKey) — run the claim action + status + refetch (the claimed account disappears). A sweep control (after grace) if applicable. Delphi + a11y.
- Headless-render the payout states. Green + RUN the gated claims E2E. Commit `feat(app): claim/close/sweep payout flows + gated E2E`.

### RF3 — Create-oracle (new page + action + gated E2E)  [parallel with RF1]
- **Action** (`app/src/data/actions/create.ts`): `buildCreateOracleIxs({ connection, nonce, question|promptHash, optionsCount, deadline, creator, kassMint, usdcMint, ... })` — hash the question text → promptHash (`hashToContentHash`), derive the creator's KASS token account (+ idempotent-create), assemble `createOracle`. Nonce: derive the next free nonce (or accept one; the protocol's oracle-count or a random nonce — confirm how nonces are allocated). Validation (optionsCount 2..N, deadline in the future, question non-empty).
- **A create page** (`app/src/pages/CreateOracle.tsx` at `/oracles/new`, linked from `/oracles`): a Delphi form — question text, options count, deadline (a date/time → unix; hand-roll, no date lib), the mints (default the protocol's KASS/USDC or let the user paste). On submit → wallet sender + status + on success navigate to the new `/oracles/:pubkey`. Gated on connected. **This is a new page — independent of OracleDetail (no collision with RF1/RF2/RF4).**
- **Offline unit tests** + **gated surfpool E2E** (`app/test/create.e2e.test.ts`): initProtocol, then `buildCreateOracleIxs` + send via keypair → assert a real Oracle on-chain (decode: promptHash matches sha256(question), optionsCount, deadline, creator). 
- Headless-render the create form states. Green + RUN the gated create E2E. Update the `/oracles` page with a "Create oracle" CTA. Commit `feat(app): create-oracle page + action + gated E2E`.

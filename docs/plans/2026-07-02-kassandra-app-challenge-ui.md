# Kassandra dApp — Challenge-Market Trading UI (Milestone B) — Design + Plan

> **For Claude:** REQUIRED SUB-SKILL: subagent-driven-development (per-task implement + review).

**Goal:** Turn the thin JSON-paste challenge surface into a real challenge-market UI: **live market visualization** (pass/fail prices, TWAP→margin progress, twapEnd countdown, challenger position), **trade/crank/settle an OPEN market** (swap the conditional AMMs + crank the TWAP + settle-when-ready), and **client-side market composition** (open a challenge from the browser without pasting runner JSON — question → 2 conditional vaults → split → 2× createAmm → addLiquidity → openChallenge). All Delphi. NO core (programs/runner/sdk) change (add app-side decoders/builders that CONSUME the SDK; if a genuine SDK gap appears, STOP-and-report — don't patch sdk-src). Proven by gated forked-mainnet E2Es.

## Feasibility (from recon — all achievable without SDK change)
- **Amm decoder:** the SDK has NO v0.4 `Amm` decoder (only builders + PDAs). The app adds `decodeAmmV04` using the authoritative offsets in `programs/kassandra/src/cpi/metadao.rs:145-178` (disc `8ff5c8114ad6c487`; base_amount@115, quote_amount@123, oracle.last_updated_slot@131, oracle.aggregator u128@171, created_at_slot@9, start_delay_slots@219, AMM_MIN_LEN 227) — mirror the test-local `decodeAmmTwap` (`sdk/test/surfpool/challenge-market-e2e.test.ts:863-879`). Price = quote/base; TWAP = `aggregator/(last_updated_slot − (created_at + start_delay))` scaled by PRICE_SCALE 1e12 (the settle math, `metadao.rs:153-156`).
- **Market** already surfaced by `app/src/data/oracles.ts` `fetchOracleDetail` (passAmm/failAmm/kassVault/usdcVault/twapEnd/challengerUsdc/settled + escrow/holders).
- **Trade/crank/settle an open market:** `ammV04.swap` (`sdk/src/amm-v04/instructions.ts:201`), `ammV04.crankThatTwap` (`:246`), existing `buildSettleChallengeIxs` (`app/src/data/actions/challenge.ts:202`) — all accounts derive from the decoded Market + amm-v04 PDAs + user ATAs. crank rate-limited 1/150 slots; settle gated on `now/slot ≥ twapEnd`.
- **Compose:** the primitives all EXIST — `futarchy.initializeQuestion` (`sdk/src/futarchy/instructions.ts:129`), `futarchy.initializeConditionalVault` (`:158`, creates the vault + conditional mints), `futarchy.splitTokens` (`:222`), `ammV04.createAmm`/`addLiquidity`. Disc-verified against the challenge-market-e2e's real CPIs (`composeQuestion`/`composeVault` `:426-504`). The heavy part = the multi-tx orchestration + liquidity-seeding + observation/decimals math (mirror `composeMarket`/`buildPool` `:639-829` for the REAL bits; the E2E's setAccount pieces [oracle-owned holders, initial balances] become ordinary ATA-creates + split in production).

## CU1 delta (implemented)
- **`app/src/data/ammV04.ts`** (pure, no React): `decodeAmmV04(data)` — guards the 8-byte disc `8ff5c8114ad6c487` + `>= AMM_MIN_LEN` (227), reads base/quote mint (Pubkey @49/@81), base/quote decimals (u8 @113/@114), base/quote amount (u64 @115/@123), created_at/last_updated/start_delay slots (u64 @9/@131/@219), aggregator (u128 @171) — offsets exported as consts, mirroring `cpi/metadao.rs:145-178` + the test-local `decodeAmmTwap`. Throws a typed `AmmDecodeError` on wrong disc/short buffer. Helpers: `instantaneousPrice` (decimals-aware quote/base as a Number ratio, raw reserves kept as bigints, null on empty base), `twapPrice` (`aggregator/(last_updated−(created+start_delay))`, PRICE_SCALE-scaled bigint, null pre-start-delay / zero-aggregator — mirrors `get_twap()`/settle), `marginProgress(failTwap, passTwap, marginNum, marginDen)` = `(fail−pass)·den/(pass·num)` (0 at no divergence, 1 at the margin, >1 once FAIL clears it → would disqualify; guards nulls + zero/absent pass), and `fetchMarketAmms(connection, market)` (getAccountInfo passAmm/failAmm → decode, null on absent/undecodable).
- **`app/src/hooks/useMarketAmms.ts`**: `useMarketAmms(market)` → `{ pass, fail, loading, refetch }`. Same `useEffect`+guard shape as `useOracles` (no TanStack), manual refetch + a 15 s poll for a live unsettled market (no websocket); `?mock`/`VITE_MOCK` resolves `mockMarketAmms()` offline; null market ⇒ no fetch.
- **`app/src/components/oracles/ChallengeMarketPanel.tsx`**: read-only viz rendered ADDITIVELY under the existing `MarketCard` in the detail's Challenge-market section — pass vs fail spot price + TWAP (decimals-aware; "TWAP forming…" when null), a flat divs+tokens TWAP→margin bar (the **single ember accent** lights at `progress ≥ 0.85`), a `relativeDeadline(twapEnd)` countdown ("settles in …" / "Window closed"), the challenger's USDC (groupDigits), and each pool's raw reserves (groupDigits). Settled ⇒ a calm confirmed chip.
- **`app/src/data/mockOracles.ts`** (ADD-only): `encodeMockAmm` writes real `Amm` byte blobs at the pinned offsets; `mockMarketAmms()` decodes pass (twap 1.000 / spot 1.000) + fail (twap 1.090 / spot 1.090) pools ⇒ margin progress 0.90 (near-margin ember) for the challenged fixture. RU1/RU2 fixtures untouched.
- **`app/test/ammV04.unit.test.ts`** (offline): `decodeAmmV04` over a hand-built blob (every field + full u128 aggregator; wrong-disc + short-buffer throw), `instantaneousPrice`/`twapPrice`/`marginProgress` on known inputs incl. the pre-start-delay/zero-aggregator → null guard + the near-margin (0.90) and at/over-margin cases. 11 test files / 116 tests green.
- **Verify:** `pnpm --filter app typecheck` + `test` (116) + `build` (verify-css OK) + `lint` all green; headless-rendered `/oracles/<challenged>?mock` (cached chromium) — the panel shows pass/fail spot+TWAP, the 90% ember margin bar, "settles in 12h", 500,000,000 challenger USDC, and both pools' reserves; ONE ember accent from the panel (the margin bar), one h1, 0 console errors, existing sections/`MarketCard`/`ChallengeControl` intact.
- **Decisions:** (1) `marginProgress` uses the DIVERGENCE form `(fail−pass)·den/(pass·num)` — 0 at no divergence — not the raw ratio `fail·den/(pass·(den+num))` which reads a misleading ~0.91 at fail==pass; both cross 1 at exactly the same disqualify threshold, so it's equivalent to settle but a truer progress bar. (2) `twapPrice` returns the PRICE_SCALE-scaled integer verbatim (no extra ×PRICE_SCALE — the aggregator is already scaled), mirroring `decodeAmmTwap`/settle exactly. (3) Live AMMs are surfaced via the dedicated `useMarketAmms` hook (not folded into `fetchOracleDetail`) to keep the read layer simple. (4) The panel's single ember is confined to the margin bar; the header chip stays quiet so there is exactly one ember region added.

# Kassandra dApp ‚Äî Write Flows: Participation Trio (slice 3) ‚Äî Design + Plan

> **For Claude:** REQUIRED SUB-SKILL: subagent-driven-development (per-task implement + review).

**Goal:** Add the first WRITE flows to the dApp: **propose** (propose an answer + bond), **submitFact** (submit evidence + stake), **voteFact** (approve/duplicate-vote on a fact + stake) ‚Äî on EXISTING oracles, signed by the connected wallet. Build a reusable **tx send-and-confirm layer** (a signer-abstraction seam: wallet-adapter's `sendTransaction` for the UI, a keypair for tests) + pure **action builders** (assemble the SDK ixs + derive/ensure the authority's KASS ATA), wire them into the oracle detail page as forms with pending/confirming/success/error status UX, and prove the action layer against a **seeded surfpool** via a funded keypair. All in the Delphi visual language.

## Decisions (locked with the user)
1. **Participation trio** (propose + submitFact + voteFact) on existing oracles. NO create-oracle form (later slice); finalize/challenge/claim also later.
2. **Proof:** actions = pure ix-builders + a `send(signer)` seam. A gated surfpool E2E driven by a funded KEYPAIR seeds an oracle, drives each action, and asserts the on-chain result (a real Proposer / Fact / FactVote created). The FORMS are headless-rendered (validation, disabled-when-disconnected/wrong-phase, pending/success/error) with a mock signer. The UI wraps the same action layer with wallet-adapter `sendTransaction`. (A browser wallet can't be automated ‚Äî this proves the real write path + the form UX without one.)
3. Builds on slice 1 (Delphi) + slice 2 (wallet/provider/shell + the read data layer + oracle detail page). **Build the SDK first.** Read paths stay untouched.

## Source of truth
- **SDK write builders** (return `TransactionInstruction`, take the signer as an address + its KASS token account): `sdk/src/instructions/lifecycle.ts` `propose(args: {oracle, authority(signer), authorityKass, option u8, bond u64})`; `sdk/src/instructions/dispute.ts` `submitFact(args: {oracle, submitter(signer), submitterKass, contentHash[32], stake u64, uri<=200})` + `voteFact(args: {oracle, fact, voter(signer), voterKass, kind u8, stake u64})`. `sdk/src/constants.ts`: `VOTE_APPROVE`/`VOTE_DUPLICATE` (the voteFact `kind`), `Phase` enum, `KASSANDRA_PROGRAM_ID`, `Ix`. `sdk/src/pda.ts` (proposer/fact/factVote/stakeVault derivers ‚Äî the builders call these internally). NOTE each action needs the authority's **KASS ATA** ‚Äî the app derives it (ATA of the wallet pubkey + the oracle's `kassMint`, which the read layer already decodes) and PREPENDS an idempotent create-ATA ix if it's missing.
- **Phase gating** (`Phase` enum, `sdk/src/constants.ts` + the read layer's decoded `oracle.phase`): propose is valid only in the **Proposal** phase; submitFact + voteFact only in the **Fact** phase(s) (FactProposal / FactVoting ‚Äî confirm the exact phase names + which action each allows from the enum/state.rs). The forms must gate on the current phase + wallet-connected; the E2E advances the seeded oracle into the right phase for each action (mirror how the SDK E2Es use `advancePhase`).
- **Slice-2 to build on:** `app/src/lib/cluster.ts` (`useConnection`/`useCluster`), `app/src/data/oracles.ts` (`fetchOracleDetail` ‚Äî refetch after a successful write), `app/src/pages/OracleDetail.tsx` (add the action forms here), `app/src/components/oracles/*` + `app/src/components/ui/*` (Delphi primitives), `app/src/hooks/useOracles.ts` (the useAsync + refetch pattern). The wallet: `@solana/wallet-adapter-react` `useWallet()` ‚Üí `publicKey`, `sendTransaction(tx, connection)`, `connected`.
- **Send/confirm pattern:** the SDK's `v0.ts` `confirmSignature(connection, sig)` (poll getSignatureStatuses) is reusable for confirm; for legacy sends the wallet's `sendTransaction(new Transaction().add(...ixs), connection)` returns a signature. The E2E's keypair signer path mirrors `sdk/test/surfpool/harness.ts` (`sendIx`/confirm with a Keypair).
- **KASS token dependency:** propose/submitFact/voteFact all move KASS (bond/stake) from the authority's ATA. The connected wallet must hold KASS + have the ATA. Forms: show the KASS balance, disable + explain if insufficient/absent, prepend create-ATA if the ATA is missing (idempotent). The E2E mints KASS to the keypair + ensures the ATA (harness does this).

## Tasks

### WF1 ‚Äî Write action layer + send-and-confirm seam + gated surfpool E2E
- **Send-and-confirm seam** (`app/src/data/send.ts` ‚Äî pure, no React): a `Signer` abstraction that both a wallet and a keypair satisfy, e.g. `type TxSender = (ixs: TransactionInstruction[]) => Promise<string>` (returns the signature) ‚Äî the UI supplies a wallet-backed sender (`sendTransaction`), the test supplies a keypair-backed sender. Plus `sendAndConfirm(connection, sender, ixs): Promise<{ signature: string; confirmed: true }>` that sends then confirms (reuse the SDK's `confirmSignature` or poll `getSignatureStatuses`; surface a failed tx / preflight error as a typed `SendError` with the program log). Handle blockhash/lastValidBlockHeight where the legacy send needs it.
- **Action builders** (`app/src/data/actions.ts` ‚Äî pure, take a `Connection` for ATA-existence checks, NO React): 
  - `buildProposeIxs({ connection, oracle, oracleAcct, authority, option, bond }): Promise<TransactionInstruction[]>` ‚Äî derive the authority's KASS ATA (from `oracleAcct.kassMint`); if the ATA is missing, prepend an idempotent create-ATA ix; then the SDK `propose({ oracle, authority, authorityKass, option, bond })`. 
  - `buildSubmitFactIxs({ connection, oracle, oracleAcct, submitter, contentHash, stake, uri })` ‚Äî same ATA handling + SDK `submitFact`. (contentHash: accept a hex string or compute `sha256(uri-content)`? For slice 3 accept a user-supplied 32-byte hex OR hash a pasted string ‚Äî the form decides; the builder takes the 32 bytes.)
  - `buildVoteFactIxs({ connection, oracle, oracleAcct, fact, voter, kind, stake })` ‚Äî same ATA handling + SDK `voteFact`.
  - Each returns the full ix list (create-ATA? + the action ix). Validate inputs (bond/stake > 0, option < optionsCount, uri <= 200 bytes) with typed errors the form surfaces.
- **Offline unit tests** (`app/test/actions.unit.test.ts`, default suite): with a mock `Connection` (ATA present vs absent), assert each builder produces the right ix list (create-ATA prepended only when absent; the action ix's data/keys match the SDK builder for known inputs), and input validation rejects (bond 0, option out of range, uri too long). No network.
- **Gated surfpool E2E** (`app/test/writes.e2e.test.ts`, `KASSANDRA_E2E=1`): boot surfpool + deploy + initProtocol + seed an oracle (reuse `sdk/test/surfpool/harness.ts`); mint KASS to a funded test KEYPAIR + ensure its ATA. Then, driving the WF1 action layer with a KEYPAIR-backed sender: (a) in Proposal phase ‚Üí `buildProposeIxs` + send ‚Üí assert a real **Proposer** account exists on-chain (decode it: authority == the keypair, option, bond); (b) advance to the Fact phase ‚Üí `buildSubmitFactIxs` + send ‚Üí assert a **Fact** exists (contentHash, stake, uri); (c) `buildVoteFactIxs` (approve) + send ‚Üí assert a **FactVote** exists (voter, kind, stake). This proves the real write path end-to-end via the action layer. (Reuse the read layer's `fetchOracleDetail` to assert, or decode directly.)
- `pnpm --filter @kassandra/sdk build` (SDK first) + `pnpm --filter app typecheck` + `test` (offline unit green) + `build` (verify-css OK) + `lint`. RUN the gated `KASSANDRA_E2E=1 pnpm --filter app test` (the writes E2E ‚Äî confirm each action lands on-chain). Commit `feat(app): write action layer + send/confirm seam + gated surfpool write E2E`.

### WF2 ‚Äî Write UI: propose / submit-fact / vote-fact forms on the oracle detail page
- Add the three actions to `app/src/pages/OracleDetail.tsx` (or `app/src/components/oracles/actions/*`), each a Delphi form/panel gated on `connected` + the correct `oracle.phase`:
  - **Propose** (Proposal phase): an option selector (0..optionsCount-1, labeled) + a bond amount (KASS) input ‚Üí `buildProposeIxs` + the wallet sender.
  - **Submit fact** (Fact phase): a content-hash input (a 32-byte hex field, OR a "hash this text/uri" helper) + a uri input (<=200 bytes) + a stake amount ‚Üí `buildSubmitFactIxs`.
  - **Vote fact** (Fact phase): on each Fact card in the detail's Facts section, an Approve / Duplicate vote control + a stake amount ‚Üí `buildVoteFactIxs` for that fact.
  - Each form: client-side validation (amount > 0, within KASS balance, option/uri bounds) with inline Delphi error text; a submit button (PrimaryChestnut) that runs the action ‚Üí wallet `sendTransaction` ‚Üí confirm, with **status UX**: idle ‚Üí "Signing‚Ä¶" (wallet prompt) ‚Üí "Confirming‚Ä¶" (a tx-status inline indicator or a small toast) ‚Üí success ("Proposed ¬∑ <sig short>", link to the sig on the cluster's explorer if not localnet) OR error (the typed SendError / program log, human-readable). On success, **refetch** the oracle detail (the new proposer/fact/vote appears) + reset the form.
  - **Gating + affordances:** disabled + a clear reason when disconnected ("Connect a wallet to propose"), wrong phase ("Proposing is closed ‚Äî this oracle is in the Fact phase"), or insufficient KASS (show the balance; "You need KASS to bond"). Read-only browsing is unaffected when disconnected.
- **A tiny toast/status system** if not present (a minimal Delphi toast context, or inline per-form status) ‚Äî keep it on-brand (parchment/sepia, ember only for a genuine error accent, chestnut for the action). No heavy shadows.
- Delphi-faithful (chestnut-only fill, flat + hairlines, serif ‚â•20px, radii vocabulary, ember sparingly), responsive (375/1280), accessible (labeled inputs, aria on status, focus-visible, error text associated with inputs, one h1/page unchanged). 
- **Render verification:** headless-render the detail page's action forms (via the mock affordance / a mock signer) at 375 + 1280 in the states: disconnected (gated), connected-idle, submitting, success, error ‚Äî confirm 0 console errors + the Delphi look + the gating. (Can't automate a real wallet; render the form STATES with a mock signer + a mock connected pubkey.)
- `pnpm --filter app typecheck` + `test` (offline) + `build` (verify-css OK) + `lint` green. Update `app/README.md` (the write flows, the KASS requirement, the wallet-signing UX, slice-3-vs-next). Commit `feat(app): propose / submit-fact / vote-fact forms (wallet-signed) on oracle detail`.
